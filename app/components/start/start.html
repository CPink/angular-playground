Getting Started

As you continue throughout you will see terminology that you may or may not of encountered in the past.

Glossary of Terms

  • Angular - A JavaScript Framework built by a team of googlers
  • ng - An abbreviation of the work Angular, Also a common prefix for Directives
  • Directives - A directive is usually a tag, html element, or element attribute that comes with predefined instructions
  • Controllers - Controllers are the JavaScript section tied to components or views. They are the driving force behind the functionality of site
  • View - The view is the html that gets rendered by the browser. The view often has a mix of ng expressions
  • Expressions - ng expressions are defined with a double pair of curley braces {{}} where anything inside the curley braces will be evaluated as javascript
  • Bindings - A binding is a connection between a property or function in javascript and your view
  • Services - Services are reserved for code that is shared amongst controllers and views. They are also the primary outlet for communicating with other applications usually through a restfull API

Some of the Basics

Setting up an angular app isn't necessarily difficult but it is a bit involved. For that reason the Playground is entierly setup for you. Learning how to use Angular is much more important then knowing the ins and outs of spinning up an app from scratch. But here are a few key points.

  • app.js

    			
//Initializing the Angular App and giving it a name
var app = angular.module('The-Name-Of-Your-Application', []);

/* DON'T WORRY ABOUT THIS RIGHT NOW
* The second argument is an array where at some point
* you can specify other angular modules that your app
* is dependent upon.
*/

Once you have the app initialized lets tie it to the HTML this is a two part step

  • index.html

    			
<!--using the ng-app directive we tie our angular app to our html-->
<html ng-app="The-Name-Of-Your-Application">
<head>
</head>
<body>

	<!--Your HTML...-->

	<!--Notice you must load angular before loading app.js-->
	<script src="path-to-angular.js"></script>
	<script src="app.js"></script>
</body>
</html>

Now that your app and your html are tied together they know how to refrence one another. This has essentially created a two way street between your view and the JavaScript code that is going to drive it or make the page more dynamic

To add some extra functionality and organization to our page we can setup individual components or mini-views within our page.

To keep it simple for now think of needing to register angular components or controllers to your main application to do this in javascript we specify the following

  • controller-name.js

    
//remember app was already setup in app.js and is referencing the app that was registered or tied to your view with the ng-app directive
app.controller('The-Name-Of-A-Controller', functionToExecute);

function functionToExecute(){
  //your code here
}

  • controller-name.js

    
//By convention with Angular Controllers are CamelCased and are essentially Constructor functions
app.controller('MainController', MainController);

//Also by convention controllers usually end with the word Controller  
function MainController(){
  //We are now in a constructor function so anything we put on this will be accessible to us in our view
  
  this.name = "The Angular PlayGround";
  
}

  • index.html

    
<!--There are a few ways of adding a controller into your view for now we will use another directie called ng-controller -->
<div ng-controller="MainController as controller">
	<!--Anything in the scope of this div now has direct access to our MainController object wich we can easily access by using the alias setup with the as keyword in the ng-controller directive -->
	{{controller.name}}
</div>
